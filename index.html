Buenas prácticas en HTML: una guía breve

Por Equipo de Desarrollo — 19 de febrero de 2026

Introducción

HTML es la base de cualquier sitio web. Utilizarlo correctamente mejora la accesibilidad, el rendimiento y la mantenibilidad. Este artículo recoge prácticas recomendadas para escribir HTML claro y robusto.

Principios claves

  • Semántica: usa etiquetas como <header>, <nav>, <main>, <article>, <section> y <footer> para describir la estructura del contenido.
  • Accesibilidad: añade atributos ARIA cuando sea necesario, proporciona textos alternativos (alt) para imágenes y usa roles apropiados.
  • Orden lógico de encabezados: usa <h1>–<h6> según la jerarquía, sin saltos arbitrarios.
  • Formularios accesibles: etiqueta los controles con <label> y usa atributos como aria-label cuando el label visual no sea posible.
  • Enlaces descriptivos: evita textos genéricos como "haz clic aquí"; describe el destino del enlace.
  • Separación de responsabilidades: evita estilos en línea y scripts mezclados en el marcado; usa archivos CSS/JS externos.

Rendimiento y validación

Valida tu HTML con herramientas como el validador de W3C, optimiza imágenes, carga recursos de forma diferida cuando sea posible y minimiza el tamaño de los archivos para mejorar tiempos de carga.

Resumen: escribe HTML semántico, accesible y mantenible. Pequeños hábitos producen grandes mejoras.
